AWS Setup

Configure your AWS account correctly before writing a single line of application code. Getting this right the first time prevents confusing permission errors in every chapter that follows.

⏱ 45–60 min 💰 $0 this chapter 🔧 AWS Console + Terminal ✅ Verified at the end
🎯
Goal: Have a fully configured AWS account with Bedrock access, an IAM user, DynamoDB table, S3 bucket, and AWS CLI working on your laptop — all verified with a test script before moving on.
🧱
Deliverable: A verify_aws.py script that prints ✅ for every check. Until all checks pass, do not move to Chapter 3.

What We Are Setting Up

Five AWS services, each with a specific role in the Neuron platform. We configure all of them now so they are ready when the code needs them.

Setup Flow Overview

Ten steps in total. Steps 1–4 are done in the AWS Console (browser). Steps 5–10 are in your terminal. Follow them in order — each step depends on the previous.

Steps 1–4 — In the AWS Console

Step 1 — Create AWS Account
Sign up at aws.amazon.com

If you already have an AWS account, skip to Step 2.

  1. Go to aws.amazon.com → click Create an AWS Account
  2. Enter your email and choose an account name (e.g. neuron-platform)
  3. Enter credit card details — nothing in this chapter costs money
  4. Choose the Basic (Free) support plan
  5. Wait for the confirmation email, then sign in
🔴
Do this immediately after creating your account: Enable MFA on your root account. Console → top-right account name → Security credentials → Multi-factor authentication → Assign MFA device. Never use the root account for day-to-day work.

Step 2 — Enable Amazon Bedrock Models
Request Claude access in the AWS Console

AWS Bedrock does not give you Claude access by default — you must request it.

  1. Sign in to console.aws.amazon.com
  2. Search Bedrock in the top bar → click Amazon Bedrock
  3. Set region to us-east-1 (top-right dropdown) — use this region throughout
  4. Left sidebar → Model access
  5. Click Modify model access (orange button, top right)
  6. Check these three models:
ModelUsed ForCost (approx)
Claude Sonnet 4
anthropic.claude-sonnet-4...		Main model — all agent reasoning$3 / 1M input tokens
Claude Haiku 4
anthropic.claude-haiku-4...		Cheap fast tasks — routing, summarisation$0.25 / 1M input tokens
Titan Embeddings V2
amazon.titan-embed-text-v2...		Knowledge Base RAG (Chapter 20)$0.02 / 1M tokens
⏱
After clicking Save, wait 2–5 minutes and refresh. Status changes from "Available" → "Access granted". Do not proceed until you see this.
Step 3 — Create IAM User
Create neuron-dev-user with the right permissions
  1. In AWS Console → search IAM → click Users → Create user
  2. Username: neuron-dev-user → Next
  3. Select Attach policies directly
  4. Search and check these four policies:
Policy NameWhy We Need It
AmazonBedrockFullAccessCall Claude via the Bedrock API
AmazonDynamoDBFullAccessRead/write conversation sessions
AmazonS3FullAccessUpload documents for Knowledge Base
AmazonECR_FullAccessPush Docker images (Chapter 14)
Step 4 — Create Access Keys
Generate credentials for your laptop
  1. Click neuron-dev-user in the users list
  2. Click the Security credentials tab
  3. Under Access keys → Create access key
  4. Select Local code → Next → Create access key
  5. Download the CSV file NOW — you cannot see the secret key again after this screen
🔴
Never commit these keys to GitHub. They give full access to your AWS account. Store them in 1Password, macOS Keychain, or a secure note. We add them to a .env file which is always in .gitignore.

Steps 5–9 — In Your Terminal

Step 5 — Install and Configure AWS CLI
Connect your terminal to AWS

The AWS CLI lets you control AWS from your terminal — creating tables, pushing images, and checking costs.

bash Install AWS CLI 
# macOS
brew install awscli

# Ubuntu / Debian
sudo apt-get install awscli

# Verify installation
aws --version
# Expected: aws-cli/2.x.x Python/3.x.x

# Enter when prompted:

Step 6 — Install uv (Python Package Manager)
The modern replacement for pip — used in every chapter

We use uv instead of pip throughout this guide. It is 10–100× faster, creates reproducible builds, and manages Python versions automatically.

bash Install uv 
# macOS / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh

# Windows (PowerShell)
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"

# Verify
uv --version
# Expected: uv 0.x.x
Step 7 — Create DynamoDB Table
The database that stores all conversation history

DynamoDB is AWS's serverless database. "Serverless" means you pay only for what you use — at our scale, the cost is essentially $0 per month. No maintenance, no servers to manage.

bash Create the neuron-sessions table 
# Create the sessions table
aws dynamodb create-table \
  --table-name neuron-sessions \
  --attribute-definitions AttributeName=session_id,AttributeType=S \
  --key-schema AttributeName=session_id,KeyType=HASH \
  --billing-mode PAY_PER_REQUEST \
  --region us-east-1 \
  --profile neuron-dev

# Enable TTL — auto-delete sessions after 24 hours
aws dynamodb update-time-to-live \
  --table-name neuron-sessions \
  --time-to-live-specification "Enabled=true,AttributeName=ttl" \
  --region us-east-1 \
  --profile neuron-dev

# Verify table is ACTIVE
aws dynamodb describe-table \
  --table-name neuron-sessions \
  --region us-east-1 \
  --profile neuron-dev \
  --query "Table.TableStatus"
# Expected: "ACTIVE"

Step 8 — Create S3 Bucket
Object storage for documents and Docker image layers

The S3 bucket holds documents for the Bedrock Knowledge Base (Chapter 20). We create it now so the name is reserved.

Replace YOUR_ACCOUNT_ID with your 12-digit AWS account ID. Run aws sts get-caller-identity --query Account --output text --profile neuron-dev to find it.

bash Create S3 bucket with account ID suffix for uniqueness 
# Get your account ID
ACCOUNT_ID=$(aws sts get-caller-identity \
  --query Account --output text --profile neuron-dev)

# Create bucket (name must be globally unique — account ID ensures this)
aws s3 mb s3://neuron-platform-docs-$ACCOUNT_ID \
  --region us-east-1 \
  --profile neuron-dev

# Verify
aws s3 ls --profile neuron-dev | grep neuron-platform-docs
# Expected: 2024-xx-xx xx:xx:xx  neuron-platform-docs-123456789012

What You Have Learned

Amazon Bedrock
AWS's AI model hosting service. Claude lives here. You pay per token, AWS handles the infrastructure, and your data stays inside AWS.
IAM — Least Privilege
Always use the least permissions needed. Root account only for billing. IAM user for local dev. IRSA for production pods — no credentials stored.
DynamoDB is Serverless
No servers, no maintenance, pay per use. At small scale it's essentially free. TTL auto-expires old sessions so you never hit storage limits.
uv over pip
uv manages Python versions, virtual environments, and dependencies. 10–100× faster than pip. We use it in every single chapter — learn it once.
Verify before building
A working foundation saves hours of debugging later. The verification script catches permission errors, typos, and missing resources before they cause confusing failures in later chapters.
Environment variables
Secrets go in .env files, never in code. .env is always in .gitignore. This is non-negotiable — one leaked key can cost thousands in AWS bills.
