The Five Mental Models

Before writing a single line of code, you need these five mental models in your head. Each one explains a different aspect of how Neuron works.

Figure 1.1 — The Five Mental Models at a Glance

Mental Model 1 — Neuron is "FaaS for AI Agents"

Think of AWS Lambda. You submit a function, it runs in a container, you get a result back. Neuron is the same idea — but instead of a function, you submit a plain-English prompt, and the AI agent figures out what tools to call and in what order.

AWS LambdaNeuron
You submit: { code }You submit: "Generate tests for PR #234"
Runs in a containerAI Agent thinks, plans, calls tools
Returns: resultReturns: tests + Jira ticket + Slack message
You define the logicAI defines the logic from your prompt

Mental Model 2 — MCP is the USB Standard for AI Tools

Before USB, every device (keyboard, mouse, printer) had a different connector. USB created one universal standard that works with everything. MCP (Model Context Protocol) does the same for AI tools.

Figure 1.2 — Why MCP Changes Everything

Mental Model 3 — The Brain vs The Hands

This is the most important distinction in the entire platform. The AI agent is the brain — it thinks, reasons, and decides. The MCP tools are the hands — they execute, call APIs, and return data. Never mix these two responsibilities.

Figure 1.3 — Brain vs Hands: Strict Separation of Concerns
⚠️
The golden rule: The MCP tools are dumb. jira_create_issue() just calls the Jira API. It has zero intelligence. All the intelligence — which tool, when, with what parameters — lives entirely in the AI Agent. Never put business logic in a tool.

Mental Model 4 — Three Repos, Each With One Job

Neuron is split into three repositories. Each has exactly one responsibility. This separation means you can upgrade, replace, or scale any layer without touching the others.

Figure 1.4 — Three-Repository Architecture
