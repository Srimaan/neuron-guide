The Key Tool: Rich Messages with Block Kit

Plain text messages are fine for simple updates. But for sprint reports, incident summaries, and release notes, Slack's Block Kit delivers formatted messages with sections, dividers, bold text, bullet lists, and contextual footers. The AI generates the Block Kit JSON from your plain-English description.

Required Bot Scopes

Each Slack tool needs specific OAuth scopes. You add these to your Slack App and reinstall it to your workspace. Without the right scopes, tool calls return a permission error.

Build Step

Step 10.1 — Update Slack App Scopes
Add the required Bot Token Scopes before writing new tools
  1. Go to api.slack.com/apps → select your Neuron app
  2. Click OAuth & Permissions in the left sidebar
  3. Under Bot Token Scopes add: chat:write, chat:write.public, channels:history, channels:read, groups:history, groups:read, search:read, users:read, users:read.email, reactions:write
  4. Click Reinstall to Workspace — this generates a new token with the new scopes
  5. Copy the new xoxb-... token and update your .env and docker-compose.yml

Claude Prompt:

"I am expanding neuron-mcp-server's Slack tools. Existing: slack_send_message. Tech stack: Python 3.11, FastMCP, slack-sdk WebClient, utils.py with slim_result() and format_error().

Add these 7 async tools to slack_tools.py, each with @mcp.tool(), docstring, type hints, try/except with format_error():

1. slack_send_rich_message(channel:str, text:str, blocks:list) → dict — sends a Block Kit message. blocks is a list of block dicts. Returns {ts, channel, permalink}. text is the fallback for notifications.
2. slack_search_messages(query:str, count:int=20) → dict — uses client.search_messages(query=query, count=count). Returns slim_result of [{ts, channel_name, text_preview, user, permalink}].
3. slack_get_channel_history(channel:str, limit:int=20) → dict — gets recent messages. Returns [{ts, user, text, reaction_count, reply_count}] slim_result max 20.
4. slack_list_channels(exclude_archived:bool=True) → dict — lists channels bot is a member of. Returns [{id, name, is_private, member_count, topic}] slim_result max 50.
5. slack_send_dm(user_identifier:str, text:str) → dict — opens DM channel then posts. user_identifier can be Slack user_id (Uxxx) or email. If email, calls users.lookupByEmail first. Returns {ts, channel}.
6. slack_add_reaction(channel:str, timestamp:str, emoji:str) → dict — adds emoji reaction to a message. emoji without colons e.g. 'thumbsup'. Returns {ok:true}.
7. slack_get_user_info(user_id:str) → dict — returns {id, name, display_name, email, status_text, is_bot, tz}.
8. slack_update_message(channel:str, timestamp:str, text:str, blocks:list=[]) → dict — edits existing message by ts. Returns {ts, channel}.

Update register_slack_tools(mcp) to register all 8 tools. Show the complete slack_tools.py."
⚠️
Invite the bot to channels first. The bot can only post to channels it is a member of. In Slack, go to each channel → Add apps → search for your Neuron app. Or use slack_list_channels() to see which channels it can already access.

What You Have Learned

Block Kit = rich formatted comms
Plain text Slack messages are fine. Block Kit messages with sections, headers, and bold text get read. The AI generates the JSON structure from your plain-English description.
Cross-integration chains matter
The power isn't in Slack tools alone — it's Jira + Slack together. Get sprint data from Jira, format it, post to Slack. One prompt, two integrations, five tool calls.
Scopes must match tools exactly
Each new tool requires specific scopes. Add them all at once and reinstall once — not one at a time. Re-installing the Slack app every time you add a tool is tedious.
Integrations complete — 30 tools
GitHub (12) + Jira (10) + Slack (8) = 30 tools. The three integrations trilogy is complete. Chapter 11 refactors all 30 down to 6 dispatcher tools — same power, 87% fewer tokens.
