Step 9.1 — Add 5 New Tools to jira_tools.py
Expand from 2 to 10 tools — sprint management, bulk create, project summary, linking
✨
Claude Prompt:

"I am expanding neuron-mcp-server's Jira tools. Existing: jira_search_issues, jira_create_issue, jira_get_issue, jira_update_issue, jira_add_comment. Tech stack: Python 3.11, FastMCP, jira library, utils.py with slim_result() and format_error().

Add these 5 tools, each async with @mcp.tool(), docstring, type hints, try/except with format_error(), slim_result() on lists:

1. jira_get_sprint(board_id:int, state:str='active') → dict — uses jira.sprints(board_id), filters by state (active/future/closed), returns {sprint_id, name, state, start_date, end_date, goal, issues_count}
2. jira_get_sprint_issues(sprint_id:int) → dict — gets all issues in sprint, returns [{key, summary, status, assignee, story_points, priority, type, url}], slim_result max 100
3. jira_bulk_create_issues(project_key:str, issues:list[dict]) → dict — each issue dict has {summary, description='', issue_type='Task', priority='Medium'}, uses jira.create_issues() for bulk creation, returns {created:[{key,url}], failed:[{summary,error}]}
4. jira_get_project_summary(project_key:str) → dict — returns {name, key, lead, open_issues_count, in_progress_count, done_this_week_count, issue_types, statuses}
5. jira_link_issues(from_key:str, to_key:str, link_type:str='relates to') → dict — common link types: 'relates to', 'blocks', 'is blocked by', 'duplicates', returns {success:true, from_key, to_key, link_type}

Update register_jira_tools(mcp) to register all 10 tools."

What You Have Learned

Bulk operations = real time savings
One bulk create call vs 5 sequential calls. This is where AI saves the most time — not one ticket, but 20 tickets from a description in seconds.
JQL is the AI's query language
The AI learns JQL patterns from your prompts and reuses them. The more specific your prompts ("all High priority bugs in ENG assigned to me"), the better the JQL it constructs.
Sprint intelligence reduces meetings
A sprint health check that used to require opening Jira, reading the board, and counting manually now takes one sentence. The AI does the counting and gives you the assessment.
Issue linking = dependency tracking
The AI can now model dependencies between work items. "ENG-234 blocks ENG-267" is something the team knows but often forgets to record. The AI records it automatically.
