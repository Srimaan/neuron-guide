The Agent Backend

Build the FastAPI service that powers your web application — the brain that receives messages, runs the Strands AI Agent, calls your MCP tools, and streams responses back word-by-word using Server-Sent Events.

⏱ 3–4 hours 💰 ~$1–3 (Bedrock) 🐍 FastAPI + Strands ⚡ SSE Streaming
🎯
Goal: Build neuron-mcp-client — a FastAPI service on port 8000 that accepts chat requests, runs a Strands Agent backed by Claude on Bedrock, calls tools on your MCP server, and streams the response via SSE.
🧱
Deliverable: POST /chat endpoint streaming real SSE events. GET /health shows MCP server connected. A curl test returns a streaming AI response that calls real tools and remembers previous messages.

Where This Fits in the Stack

After Chapter 5, two of the three services are running. The frontend (Chapter 6) plugs into this backend via HTTP. The MCP server (Chapter 3) is already running and providing tools.

The Strands Agent — Think, Act, Observe

Strands Agents is AWS's open-source Python SDK for building AI agents. It implements the think → act → observe loop — the core cycle that makes an AI agent different from a simple chatbot.

Figure 5.2 — The Strands Think-Act-Observe Loop

The SSE Streaming Protocol

Our backend doesn't send one big response at the end. It streams a series of typed events so the frontend can update the UI in real time — showing a spinner while thinking, tool names as they're called, and text as it streams.

Figure 5.3 — The Six SSE Event Types
thinking Agent is processing. Show spinner in UI. Content: "Analysing your request..."
tool_use Tool is being called. Show tool badge. Content: {"tool": "jira_search_issues", "input": {"jql": "..."}}
tool_result Tool returned data. Optional to show. Content: {"tool": "jira_search_issues", "result": [...]}
text Text chunk from Claude. Append to response in UI. Content: "You have 3 open tickets:"
complete Stream finished. Hide spinner, save history. Content: {"session_id": "abc-123", "tools_used": ["jira_search_issues"]}
error Something went wrong. Show error in UI, still ends gracefully. Content: "MCP server connection failed"

Step 5.5 — Write mcp_client/services/agent.py
The Strands Agent — the intelligence that calls tools and streams responses
✨
Claude Prompt:

"Generate mcp_client/services/agent.py with a NeuronAgent class.

__init__(self, settings: Settings): Create a Strands Agent connected to the MCP server at settings.mcp_server_url using streamable HTTP transport. Add Authorization: Bearer header if mcp_auth_token is set. Use Bedrock model settings.bedrock_model_id with system_prompt from settings.

async invoke_streaming(prompt: str, session_id: str, history: list[dict]) → AsyncGenerator[tuple[str, Any], None]: Run the agent and yield streaming tuples: ('thinking', '...') when processing, ('tool_use', {'tool': name, 'input': args}) when calling a tool, ('tool_result', {'tool': name, 'result': ...}) after tool returns, ('text', 'chunk') for each text chunk, ('complete', {'session_id': session_id, 'tools_used': list}) when done, ('error', 'message') on failure — never raise, always yield error and return.

async health_check() → bool: tests MCP server connectivity.

Include full type hints, docstrings."
Step 5.6 — Write mcp_client/routers/chat.py
The POST /chat endpoint that streams SSE events
✨
Claude Prompt:

"Generate mcp_client/routers/chat.py with a FastAPI router.

POST /chat: accepts ChatRequest, generates session_id UUID if not provided, loads history from SessionManager, calls NeuronAgent.invoke_streaming(), returns StreamingResponse with media_type='text/event-stream'. Each SSE event: 'data: {json_encoded_ChatStreamEvent}\n\n'. After stream completes save updated history. On any exception yield error event then end gracefully.

POST /chat/reset: accepts {session_id: str}, clears history, returns {success: true}.

GET /chat/history/{session_id}: returns {session_id, messages: list[ChatMessage], message_count: int} or 404.

Get agent and session_manager from request.app.state. Log each request with timing."

What You Have Learned

Strands think-act-observe loop
The agent cycles through think → act → observe automatically. You provide the model, tools, and prompt. Strands handles the loop, tool dispatch, and result injection.
SSE streaming events
Six event types: thinking, tool_use, tool_result, text, complete, error. Each tells the frontend exactly what is happening. The UI can show a spinner, tool names, and streaming text simultaneously.
FastAPI lifespan
Initialise services in the lifespan context manager — not at module level. This ensures clean startup and shutdown, and avoids import-time side effects that break testing.
Session memory via DynamoDB
The AI remembers your conversation because we save messages to DynamoDB after each turn and load them before the next. The TTL auto-deletes old sessions — no cleanup needed.
Health checks run in parallel
asyncio.gather() runs all three dependency checks simultaneously — total time is the slowest check, not the sum. Always return HTTP 200 — use the body to communicate health status.
Errors yield, never raise
In a streaming generator, an unhandled exception kills the stream mid-message. Always catch exceptions, yield an error event, then return cleanly. The frontend handles it gracefully.
