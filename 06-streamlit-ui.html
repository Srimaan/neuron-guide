The Streamlit UI

Build a real web chat interface in pure Python. Real-time streaming responses, tool-use visibility, quick action buttons, and session management — all running at http://localhost:8501.

⏱ 2–3 hours 💰 $0 🐍 Streamlit 🌐 Port 8501
🎯
Goal: Build a working web chat UI at http://localhost:8501 that streams AI responses in real time, shows which tools were called, has quick-action buttons, and manages multiple sessions.
🧱
Deliverable: neuron-frontend/ — a Streamlit app that connects to the agent backend (port 8000), streams responses word-by-word, shows tool badges under AI messages, and persists sessions across page interactions.

Streamlit lets you build a working web UI in Python in hours. You won't win design awards, but you will have something genuinely useful running the same day you start it. We upgrade to Reflex (Python-to-React) in Phase 4 — but everything you learn about state, streaming, and UX applies directly.

The Streamlit State Model

Streamlit re-runs your entire Python script from top to bottom every time a user interacts with anything. st.session_state is the only thing that persists between reruns. Understanding this is the key to writing Streamlit apps that work correctly.

How Streaming Works in Streamlit

Streamlit has st.write_stream() which accepts a Python generator and displays text progressively. We convert our SSE events into a generator that yields text chunks.

HTTP client that calls the agent backend and parses SSE events

Claude Prompt:

"I am building neuron-frontend, the Streamlit web UI for Neuron. The backend runs at BACKEND_URL env var (default http://localhost:8000).

Generate frontend/api_client.py with a NeuronAPIClient class:
1. stream_chat(prompt, session_id, user_id='web-user') → Generator[dict, None, None]: POST to {base_url}/chat, parse SSE lines, yield parsed event dicts {event_type, content}. SSE format: 'data: {json}\n\n'. Handle connection errors: yield {event_type:'error', content: str(e)}

Step 6.4 — Write frontend/app.py
The main Streamlit application — layout, state, streaming
✨
Claude Prompt:

"Generate frontend/app.py as the main Streamlit application for Neuron.

State (st.session_state):
- messages: list[dict] = [] — each {role, content, tools_used: list[str]}
- session_id: str = uuid.uuid4() as string
- processing: bool = False

Layout:
SIDEBAR: Neuron logo, session ID (first 8 chars), 'New Session' button that clears messages and generates new session_id, divider, 'Quick Actions' with 4 buttons (My open PRs / My Jira tickets / Sprint summary / Failed builds) each setting a pending prompt, divider, backend status from api_client.get_health().

MAIN AREA: Title 'Neuron AI Assistant', render all messages with st.chat_message, user messages right-aligned, assistant messages with gray background, tool badges as small st.caption under AI messages showing '🔧 tools: tool1, tool2', chat input with st.chat_input always visible at bottom.

When message submitted: set processing=True, add user message to messages, call api_client.stream_chat(), parse events: thinking → show st.spinner, tool_use → append tool name to current tools_used list, text → feed to st.write_stream generator, complete → hide spinner set processing=False show tool badges.

Use st.rerun() only when necessary. Never block the UI thread during streaming."

What You Have Learned

Streamlit reruns everything
The entire script re-runs on every interaction. st.session_state is the only persistent memory. Design your app around this constraint — not against it.
st.write_stream() + generators
Convert SSE events into a Python generator that yields text chunks. st.write_stream() handles the incremental display. Clean, simple, effective.
Tool visibility builds trust
Showing which tools the AI called under each message builds user trust. Users understand why the AI said what it said — it's not a black box anymore.
Quick actions reduce friction
The four quick-action buttons cover 80% of daily use. Users shouldn't have to type for common tasks. One click beats ten keystrokes every time.
Phase 1 is complete
Three services running locally. A real web UI. Real GitHub, Jira, Slack integration. Session memory. This is a working agentic AI platform — built in 6 chapters.
Streamlit is a stepping stone
