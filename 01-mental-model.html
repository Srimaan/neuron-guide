The Five Mental Models

Before writing a single line of code, you need these five mental models. Each explains a different aspect of how Neuron works. Read all five before continuing.

Figure 1.1 — The Five Mental Models at a Glance

Mental Model 1 — Neuron is "FaaS for AI Agents"

Think of AWS Lambda. You submit a function, it runs in a container, you get a result back. Neuron is the same idea — but instead of a function, you submit a plain-English prompt, and the AI agent figures out what tools to call and in what order.
AWS LambdaNeuron
Submit: { code }Submit: "Generate tests for PR #234"
Runs in a containerAI Agent thinks, plans, calls tools
You define the logicAI defines the logic from your prompt
Returns: one resultReturns: tests + Jira ticket + Slack message

Mental Model 2 — MCP is the USB Standard for AI Tools

Before USB every device had a different connector. USB created one universal standard. MCP (Model Context Protocol) does the same for AI tools — one standard that works with Claude, GPT, Gemini, or any future model.

Figure 1.2 — Why MCP Changes Everything

Mental Model 3 — The Brain vs The Hands

The AI agent is the brain — it thinks, reasons, and decides. The MCP tools are the hands — they execute, call APIs, and return raw data. This is the most important separation in the entire platform.

Figure 1.3 — Brain vs Hands: The Most Important Separation
⚠️
The golden rule: MCP tools are dumb executors. jira_create_issue() just calls the Jira API — no intelligence. All the intelligence lives in the AI Agent. Never put business logic inside a tool.

Mental Model 4 — Three Repos, Each With One Job

Neuron is split into three repositories, each with exactly one responsibility. This means you can upgrade, replace, or scale any layer without touching the others.

Figure 1.4 — Three-Repository Architecture

Mental Model 5 — Five Layers Like an Onion

Neuron is built in 5 layers. Each layer depends only on the one below it. You build from the inside out — Layer 1 first, Layer 5 last. Each layer can be upgraded without changing the layers above or below it.

What You Have Learned

AI agents need a body
Real tools + real credentials + real compute. Neuron provides all three. Without these, the AI can only talk.
MCP is the key abstraction
One MCP server works with Claude, GPT, Gemini, or any future model. Your tool code never changes when you change AI models.
Brain / hands separation
Intelligence in the agent, execution in the tools. Mix these and you get untestable, unmaintainable code.
Three repos, one job each
Each repo is independently replaceable. Swap Streamlit for Reflex without touching the MCP server. Loose coupling.
Build in phases
A working Phase 1 beats a half-built Phase 4 by a factor of infinity. Ship something real first.
slim_result() from day one
Context overflow is a real production problem. Cap tool responses before they cause hard-to-debug failures.
