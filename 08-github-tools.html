Step 8.1 — Add 7 New Tools to github_tools.py
Expand the GitHub tool module from 2 to 9 functions, all registered via register_github_tools()
✨
Claude Prompt:

"I am expanding neuron-mcp-server's GitHub tools. Existing: github_search_prs, github_get_pr_diff. Tech stack: Python 3.11, FastMCP, pygithub, utils.py with slim_result() and format_error().

Add these 7 new async tools to mcp_server/tools/github_tools.py, each with @mcp.tool() decorator, docstring, type hints, try/except with format_error(), slim_result() on lists:

1. github_get_file(owner:str, repo:str, path:str, ref:str='main') → dict — gets file content, decodes base64 automatically, returns {path, content, sha, size}
2. github_list_pr_files(owner:str, repo:str, pull_number:int) → dict — returns [{filename, status, additions, deletions, changes}]
3. github_create_pr_comment(owner:str, repo:str, pull_number:int, body:str) → dict — posts comment, returns {comment_id, url, created_at}
4. github_search_issues(query:str, repo:str='', state:str='open', labels:str='') → dict — searches issues, returns [{number, title, state, labels, assignees, url, body_preview}]
5. github_get_repository_info(owner:str, repo:str) → dict — returns {name, description, language, stars, forks, default_branch, topics, open_issues_count}
6. github_list_workflows(owner:str, repo:str) → dict — returns [{id, name, state, url}]
7. github_trigger_workflow(owner:str, repo:str, workflow_id:str, ref:str='main', inputs:dict={}) → dict — triggers workflow, returns {run_id, url, status:'queued'}

Update register_github_tools(mcp) to register all 9 tools. Show the complete updated file."
Step 8.2 — Update Token Scopes
Regenerate your GitHub PAT with the additional scopes needed

Go to github.com → Settings → Developer settings → Personal access tokens → Tokens (classic) → generate a new token with these scopes:

  • repo — full control of repositories
  • read:org — read org data and code search
  • workflow — trigger GitHub Actions (optional)

Update the token in your .env file and docker-compose.yml.

What You Have Learned

Code search unlocks understanding
With github_search_code, the AI gains genuine understanding of your codebase patterns — not just PR diffs. It can find all usages of a function across all files.
Chaining tools = intelligence
A single prompt triggers 3–5 tool calls automatically. The AI sequences them logically without being told the order. This is the power of the think-act-observe loop.
slim_result() is critical for search
Code search returns HTML-highlighted snippets that can be massive. slim_result() is not optional here — without it, one search overflows the context window.
Minimal permissions principle
Don't add the workflow scope unless you use the trigger tool. Each additional permission increases risk. Match scopes to the tools you actually deploy.
adding_tools is always the same
Async function + @mcp.tool() decorator + docstring + type hints + try/except + slim_result(). This pattern is identical for every tool you will ever add to Neuron.
register_*_tools() stays clean
You added 7 tools without touching main.py. The register function handles all registration. main.py stays clean regardless of how many tools you add.
